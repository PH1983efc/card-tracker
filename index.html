How Saving Works

When you toggle a card's status, the page immediately sends a request to your backend server. You need a small server running alongside this page that handles two API routes:

GET /api/read — Load all cards

Should return JSON in this shape:

{ "success": true, "rows": [["id","year","set",...], ...] }

POST /api/update — Save a toggle change

Receives this JSON body and should write the new value to your Google Sheet:

{ "id": "card-id-here", "got": true }

Quick start (Node.js / Express)

If you have Node.js installed, this is the simplest way to get a backend running:

npm install express googleapis node server.js

Your server.js needs to use the Google Sheets API with a service account. Search for "Google Sheets API Node.js quickstart" on the Google developers docs — it walks you through creating a service account and getting your credentials JSON file.

Quick start (Python / Flask)

pip install flask gspread oauth2client python server.py

Then use the gspread library to read/write your sheet. Search "gspread service account" for setup instructions.

Not technical?

The simplest option is to use Glitch.com or Replit.com — both let you run a small Node.js server for free without any local setup. You can paste a server template and connect your Google credentials there.
